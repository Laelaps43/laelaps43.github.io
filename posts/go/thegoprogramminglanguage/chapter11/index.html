在go的包下通常会存在一些以_test.go结尾的文件，他在go build阶段并不会被编译，在文件中有三种函数，一种是以Test开头的功能测试函数，运行go test会测试这些函数；一种是Benchmark开头的基准测试函数，用来测试某些操作的性能，go test会回报操作的平均执行时间；最后一种是以Example开头的示例函数。 go test会扫描包下的_test.go结尾的文件来寻找特殊的函数，使用一个临时的main包来调用这些函数，并编译和运行他们。

Test函数

test基本

在_test.go里面的Test函数有着固定的函数前面，所有的函数都要符合这个规定

func TestName(t *testing.T){

}

package word

// IsPalindrome reports whether s reads the same forward and backward.
// (Our first attempt.)
func IsPalindrome(s string) bool {
	for i := range s {
		if s[i] != s[len(s)-1-i] {
			return false
		}
	}
	return true
}

测试函数：

func TestPalindrome(t *testing.T) {
	if !IsPalindrome("detartrated") {
		t.Error(`IsPalindrome("detartrated") = false`)
	}
	if !IsPalindrome("kayak") {
		t.Error(`IsPalindrome("kayak") = false`)
	}
}

我们可以使用go test来运行测试，为当前目录，当测试成功的时候会返回PASS，测试失败会给出失败的函数等信息。

$ go test

    word_test.go:32: IsPalindrome("été") = false
--- FAIL: TestCanalPalindrome (0.00s)
    word_test.go:39: IsPalindrome("A man, a plan, a canal: Panama") = false
FAIL
exit status 1
FAIL    gopl.io/ch11/word1      0.437s

我们还可以使用-v来输出每个测试用例的名称和执行名称

$ go test -v  
=== RUN   TestPalindrome
--- PASS: TestPalindrome (0.00s)
=== RUN   TestNonPalindrome
--- PASS: TestNonPalindrome (0.00s)

$ go test -v -run="French"

如果我们需要为每一种情况写一个函数去测试，或者许多的if判断，这是可以的，但是总是差点什么，go为我们提供了一红基于表的测试（表格测试），将所有的测试都放在一个结构体中，不必去写一个个的函数。

func TestIsPalindrome(t *testing.T) {
	var tests = []struct {
		input string
		want  bool
	}{
		{"", true},
		{"a", true},
		{"aa", true},
		{"ab", false},
		{"kayak", true},
		{"detartrated", true},
		{"A man, a plan, a canal: Panama", true},
		{"Evil I did dwell; lewd did I live.", true},
		{"Able was I ere I saw Elba", true},
		{"été", true},
		{"Et se resservir, ivresse reste.", true},
		{"palindrome", false}, // non-palindrome
		{"desserts", false},   // semi-palindrome
	}
	for _, test := range tests {
		if got := IsPalindrome(test.input); got != test.want {
			t.Errorf("IsPalindrome(%q) = %v", test.input, got)
		}
	}
}

覆盖率

代码覆盖率是指在源代码中那些语句被执行所占的比例。

覆盖率准则

  • 函数覆盖率（Function coverage）：定义的函数有多少被调用
  • 语句覆盖率（Statement coverage）：有多少语句被执行
  • 分支覆盖率（Branch converage）：有多少控制结构的分支被执行
  • 条件覆盖率（Condition coverage）：有多少布尔子表达式被测试为真值和假值
  • 行覆盖率（LIne coverage）：有多少行的源代码被测试过

语句覆盖率

语句覆盖率是指在若干个测试实例中语句至少被执行一次的比例，在Go中的cover可以测试代码的语句覆盖率，但是他是被集成到test工具中。 使用go test -cover得出的为覆盖率的汇总信息。

$ go test -cover fmt
ok      fmt     0.168s  coverage: 95.2% of statements

当然我们还可以将测试的结果可视化展示出来，我们首先使用go test -coverprofile=x.out来输出测试结果，然后使用go tool cover -html=x.out来在网站中展示每行代码的覆盖率，绿色表示覆盖，红色表示未覆盖。

在go test还可以指定overmode的值：

set：某一行是否被执行（默认）
count：某一行被执行多少次
atomic：跟count一样，用在并发场景

$ go test -covermode=count -coverprofile=c.out fmt
ok      fmt     0.580s  coverage: 95.2% of statements

如果使用的是count模式，那么我们在html中展现出来的将会是，如果被执行的次数越多，那么代码的颜色就约深，反之，绿颜色约浅。

除了以html形式展示，我们还可以使用go tool cover -func=x.out生产基于函数纬度的文本可读的覆盖信息。在生成信息的最后一行包含了总的覆盖率值。

$ go tool cover -func=c.out
fmt/errors.go:22:       Errorf          100.0%
fmt/scan.go:1070:       doScan          100.0%
...
fmt/scan.go:1100:       advance         100.0%
fmt/scan.go:1181:       doScanf         97.2%
total:                  (statements)    95.2%

Benchmark函数

基准测试是在一定的工作负载下检测函数性能。

func BenchmarkIsPalindrome(b *testing.B) {
	for i := 0; i < b.N; i++ {
		IsPalindrome("A man, a plan, a canal: Panama")
	}
}

基准测试函数以Benchmark开头的函数，同时他的参数类型为*testing.B，他在一个循环中运行了N次。

$ go test -bench=.
goos: darwin
goarch: amd64
pkg: gopl.io/ch11/word2
cpu: Intel(R) Core(TM) i7-10750H CPU @ 2.60GHz
BenchmarkIsPalindrome-12         5138368               224.3 ns/op
PASS
ok      gopl.io/ch11/word2      1.833s

我们使用-bench来运行基准测试，他接受一个正则表达式，将会匹配包下的所有基准测试函数。224.3是平均5138368次的平均时间。

$ go test -bench=. -benchmem
goos: darwin
goarch: amd64
pkg: gopl.io/ch11/word2
cpu: Intel(R) Core(TM) i7-10750H CPU @ 2.60GHz
BenchmarkIsPalindrome-12         5341392               230.3 ns/op           248 B/op          5 allocs/op
PASS
ok      gopl.io/ch11/word2      1.570s

使用-benchmem可以得出运行时，所使用的内存大小和分配次数。

Example 函数

Example函数以Example开头，没有参数也没有结果，主要目的有三个：

  • 作为文档
  • 可以通过go test运行测试，通常在Example函数最后会有// Output:的注解，测试程序会将判断输出到终端的内容是否匹配注释中的文本。
  • 提供手动实验代码（不懂）。
func ExampleIsPalindrome() {
	fmt.Println(IsPalindrome("A man, a plan, a canal: Panama"))
	fmt.Println(IsPalindrome("palindrome"))
	// Output:
	// true
	// false
}
