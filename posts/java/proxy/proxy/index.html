代理

代理是一种设计模式，他用一个代理对象来控制其他的对象进行访问。在我们对对象进行访问的时候，我们不是直接访问对应的对象，而是访问他的代理对象，我们可以在访问对象之前或者之后执行对应的操作；代理提供了一种更加灵活和可控的访问方式。

代理又分为动态代理和静态代理，两种代理各有各的优缺点；但是他们都是通过一个代理对象来访问对应的被代理对象。

代理模式通常有三个角色：

  • Subject：一组可以被代理的行为的集合，通常是一个接口，保证代理对象有被代理对象有相同的行为。
  • ProxySubject：代理对象，消费端通过它来访问实际的对象。
  • RealSubject：实际被代理的对象。

静态代理

静态代理是指程序运行前，代理类已经存在于编译结果之前，也就是存在代理类的字节码（一个实际的Class文件）。

package org.example;  

// 声明一个接口为需要被代理行为的集合，Subject
public interface People {  
    void callMe();  
}

package org.example;  

// 被代理对象
public class Ron implements People{  
  
    public void callMe(){  
        System.out.println("I am Ron");  
    }  
}

package org.example;  

// People的代理对象
public class PeopleProxy implements People{  
  
    private People target;  
  
    public PeopleProxy(People target) {  
        this.target = target;  
    }  
  
    @Override  
    public void callMe() {  
        System.out.println("People Proxy Class");  
        target.callMe();  
    }  
}

public class Main {  
    public static void main(String[] args) throws Exception {  
        PeopleProxy peopleProxy = new PeopleProxy(new Ron());  
        peopleProxy.callMe();  
    }  
}

缺点：

  • 要为每一个接口实现一个代理类，如果接口中增加了方法，目标对象和被代理对象都要实现。
  • 每个代理类只有特定的功能，如果我需要别的功能，就需要再去实现一个代理类和接口，不好管理，会出现一个类有多个代理类。 优点：
  • 无序修改被代理对象。
  • 隐藏原始对象的细节，客户只要跟代理对象交互，降低系统的耦合性。

JDK动态代理

动态代理（JDK代理）的类是在程序运行时动态生成的，然后加载到JVM中，JDK是由Java本身支持的。

在JDK代理中主要由下面两个类支持：

java.lang.reflect.Proxy  
java.lang.reflect.InvocationHandler

在JDK代理中有一个限制，那就是只能为一个接口创建代理，不能为一个类创建一个代理，所有一个实现一个接口才能代理。

优点：

  • Java内置特性，使用方便
  • 动态性，动态生产代理类，不改变源代码的情况下对对象进行代理

缺点：

  • 只能代理接口，不能代理类
  • 性能较低，在代理的时候设计到反射等机制，不易频繁调用

Proxy类

这是一个JDK代理中经常会用到的一个类。

getProxyClass方法

为指定的接口创建代理类，返回代理类的Class对象，我们可以使用返回的对象来创建一个代理对象。（在新的JDK中被抛弃）

// loader 定义代理类的类加载器
// interfaces 指定需要实现的接口列表
public static Class<?> getProxyClass(ClassLoader loader,  
                                         Class<?>... interfaces)

newProxyInstance方法

创建代理类的实例对象，返回一个代理对象。

public static Object newProxyInstance(ClassLoader loader,  
                                          Class<?>[] interfaces,  
                                          InvocationHandler h)

public interface InvocationHandler {
	public Object invoke(Object proxy, Method method, Object[] args)  
        throws Throwable;  
}

InvocationHandler是一个接口，当newProxyInstance每次调用一个方法的时候，都会调用InvocationHandler接口中的invoke方法；所有我们需要代理的内容都会在这里面实现。

isProxy方法

判断指定的类是否是一个代理类

getInvocationHandler方法

获取代理对象的InvocationHandler对象

示例

package org.example;  
  
import java.lang.reflect.*;  
  
public class Main {  
    public static void main(String[] args) throws Exception {  
        People proxy = MyInvocationHandler.createProxy(new Ron(), People.class);  
        proxy.callMe();  
    }  
}  
  
class MyInvocationHandler implements InvocationHandler{  
  
    Object target;  
  
    @Override  
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {  
        System.out.println("执行方法："+method.getName());  
        Object invoke = method.invoke(target);  
        return invoke;  
    }  
  
    MyInvocationHandler(Object target){  
       this.target = target;  
    }  
  
    public static <T> T createProxy(Object target, Class<T> targetInterface){  
       if (!targetInterface.isInterface()){  
           throw new IllegalStateException("targetInterface不是接口");  
       }  
       if (!targetInterface.isAssignableFrom(target.getClass())){  
           throw new IllegalStateException("target 不继承 targetInterface");  
       }  
       return (T) Proxy.newProxyInstance(target.getClass().getClassLoader(), target.getClass().getInterfaces(), new MyInvocationHandler(target));  
    }  
}

CGLIB代理

因为JDK代理只能进行接口代理，不能进行类代理，如果需要对类进行代理的话，那么就可以使用CGLIB，CGLIB是一个强大、高性能的字节码生成库，就是说CGLIB是通过动态的去生成一个子类去覆盖需要被代理的类，那么根据Java的规则，那么他是无法代理被final修饰的方法的或者是类；在较低层，CGLIB是通过ASM（字节码操作框架）来操作字节码生成子类的。 优点：

  • 可以代理任意的接口和类，
  • 相对于JDK代理性能更加高，通过生成字节码来实现代理，避免了反射调用的开销 缺点：
  • 不能代理final方法和final类
  • 增添了项目的依赖性
// cglib依赖
<dependency>
        <groupId>cglib</groupId>
        <artifactId>cglib</artifactId>
        <version>2.2.2</version>
</dependency>

示例1：拦截所有方法

package org.example;  
  
public class Mac {  
  
    public void m1(){  
        System.out.println("M1 MacBook Pro");  
    }  
  
    public void m2(){  
        System.out.println("M2 MacBook Pro");  
    }  
}

  
public class Main {  
    public static void main(String[] args) throws Exception {  
        // 创建一个Enhancer来给某个类创建代理类  
        Enhancer enhancer = new Enhancer();  
  
        // setSuperclass来设置父类型，即需要给哪个类创建代理类  
        enhancer.setSuperclass(Mac.class);  
  
        // 设置回调，当调用代理对象的任何方法的时候，都会被MethodInterceptor接口的invoke方法处理  
        enhancer.setCallback(new MethodInterceptor() {  
            /**  
             * 代理对象方法拦截器  
             * @param o 代理对象  
             * @param method 被代理的类的方法，即Service1中的方法  
             * @param objects 调用方法传递的参数  
             * @param methodProxy 方法代理对象  
             * @return  
             * @throws Throwable  
             */  
            @Override  
            public Object intercept(Object o, Method method, Object[] objects, MethodProxy methodProxy) throws Throwable {  
                System.out.println("调用方法:" + method);  
                // 使用MethodProxy的invokeSuper调用被代理类的方法  
                Object result = methodProxy.invokeSuper(o, objects);  
                return result;  
            }  
        });  
  
        // 获取代理对象,调用enhancer.create方法获取代理对象  
        Mac mac = (Mac) enhancer.create();  
        mac.m1();  
        mac.m2();  
    }  
}

MethodInterceptor接口有个intercept方法，这个方法会拦截代理对象所有的方法调用。

示例2：拦截所有方法并返回固定值

当调用任何方法的时候都返回一个固定值。

public class Mac {  
  
    public String m1(){  
        System.out.println("M1 MacBook Pro");  
        return "M1";  
    }  
  
    public String m2(){  
        System.out.println("M2 MacBook Pro");  
        return "M2";  
    }  
}

  
public class Main {  
    public static void main(String[] args) throws Exception {  
        // 创建一个Enhancer来给某个类创建代理类  
        Enhancer enhancer = new Enhancer();  
  
        // setSuperclass来设置父类型，即需要给哪个类创建代理类  
        enhancer.setSuperclass(Mac.class);  
  
        // 设置回调，当调用代理对象的任何方法的时候，都返回系统的值  
        enhancer.setCallback(new FixedValue() {  
            @Override  
            public Object loadObject() throws Exception {  
                return "Apple M";  
            }  
        });  
  
        // 获取代理对象,调用enhancer.create方法获取代理对象  
        Mac mac = (Mac) enhancer.create();  
        System.out.println(mac.m1());  
        System.out.println(mac.m2());  
    }  
}

示例3：直接放行，不做任何操作

将NoOp.INSTANCE作为setCallback参数的时候，调用方法会直接运行。

public class Main {  
    public static void main(String[] args) throws Exception {  
        Enhancer enhancer = new Enhancer();  
  
        enhancer.setSuperclass(Mac.class);  
  
        enhancer.setCallback(NoOp.INSTANCE);  
  
        Mac mac = (Mac) enhancer.create();  
        System.out.println(mac.m1());  
        System.out.println(mac.m2());  
    }  
}

示例4：不同的方法使用不同的拦截器

通过将不同的代理方式添加进callbacks数组当中，然后将数组设置为Enhancer 的属性，最后通过CallbackFilter中的accept来判断执行那个不同的代理方式，accept方法返回callbacks数组的索引。

  
public class Main {  
    public static void main(String[] args) throws Exception {  
        Enhancer enhancer = new Enhancer();  
  
        enhancer.setSuperclass(Mac.class);  
  
        // 创建2个Callback数组  
        Callback[] callbacks = {  
                new MethodInterceptor() {  
                    @Override  
                    public Object intercept(Object o, Method method, Object[] objects, MethodProxy methodProxy) throws Throwable {  
                        Object result = methodProxy.invokeSuper(o, objects);  
                        return result;  
                    }  
                },  
                new FixedValue() {  
                    @Override  
                    public Object loadObject() throws Exception {  
                        return "Apple M2";  
                    }  
                }  
        };  
  
        // etCallbacks传递Callback数组  
        enhancer.setCallbacks(callbacks);  
        // 设置过滤器CallbackFilter  
        enhancer.setCallbackFilter(new CallbackFilter() {  
            // accept返回的Callbacks数组的索引  
            @Override  
            public int accept(Method method) {  
                String name = method.getName();  
                return name.equals("m2") ? 1 : 0;  
            }  
        });  
  
        Mac mac = (Mac) enhancer.create();  
        System.out.println(mac.m1());  
        System.out.println(mac.m2());  
    }
}

优化：

public class Main {  
    public static void main(String[] args) throws Exception {  
        Enhancer enhancer = new Enhancer();  
  
        MethodInterceptor methodInterceptor = new MethodInterceptor() {  
            @Override  
            public Object intercept(Object o, Method method, Object[] objects, MethodProxy methodProxy) throws Throwable {  
                Object result = methodProxy.invokeSuper(o, objects);  
                return result;  
            }  
        };  
        FixedValue fixedValue = new FixedValue() {  
            @Override  
            public Object loadObject() throws Exception {  
                return "Apple M2";  
            }  
        };  
  
        CallbackHelper callbackHelper = new CallbackHelper(Mac.class, null) {  
            @Override  
            protected Object getCallback(Method method) {  
                return method.getName().equals("m2") ? fixedValue : methodInterceptor;  
            }  
        };  
  
        enhancer.setSuperclass(Mac.class);  
        enhancer.setCallbacks(callbackHelper.getCallbacks());  
        enhancer.setCallbackFilter(callbackHelper);  
  
        Mac mac = (Mac) enhancer.create();  
        System.out.println(mac.m1());  
        System.out.println(mac.m2());  
    }  
}
