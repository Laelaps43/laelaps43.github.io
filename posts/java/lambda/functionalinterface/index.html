函数式接口

函数式接口（Functional interface）是有且只有一个抽象方法的接口。在java.util.function包中有许多非常通用的函数式接口。

自定义函数接口

自定义函数接口跟我们普通的定义函数接口没有任何的区别，但是我们有一个可选的@FunctionalInterface来注解这个接口，这个可以保证这个接口有只能有一个抽象方法；在javadoc中也会显示这个接口为一个函数式编接口。

@FunctionalInterface
public interface Multiply {
    int multiplyAB(int A, int B);
}

public static void main(String[] args) {
    System.out.println(multiplyAB(1, 10, (a, b) -> a * b));
}
public static int multiplyAB(int a, int b, Multiply multiply){
    return multiply.multiplyAB(a,b);
}

常用函数式接口

Supplier接口

对象实例的提供者，有一个名为get的抽象方法，没有任何参数，返回一个泛型对象；我们需要去实现get方法，让他生产一个泛型对象。

package java.util.function;

@FunctionalInterface
public interface Supplier<T> {
    T get();
}

除了Supplier 接口的使用，我们还可以指定类型，如BooleanJSupplier DoubleSupplier IntSupplier LongSupplier。 |接口名称|方法名称|方法签名|

Person person = (Person) Optional.empty().orElseGet(() -> new Person("laelaps"));  
System.out.println(person);

在Optional的orElseGet()中就要求提供一个Supplier类型的lambda函数。

public T orElseGet(Supplier<? extends T> supplier) {  
    return value != null ? value : supplier.get();  
}
接口名称 方法名称 方法签名
Supplier get () -> T
BooleanSupplier getAsBoolean () -> boolean
DoubleSupplier getAsDouble () -> double
IntSupplier getAsInt () -> int
LongSupplier getAsLong () -> long

Predicate接口

package java.util.function;

@FunctionalInterface
public interface Predicate<T> {
    boolean test(T t);
}

// 逻辑有与操作 ，对两个两个Predicate 进行与操作
default Predicate<T> and(Predicate<? super T> other) {  
    Objects.requireNonNull(other);  
    return (t) -> test(t) && other.test(t);  
}

// 逻辑非操作，对对象Predicate进行取反
default Predicate<T> negate() {  
    return (t) -> !test(t);  
}

// 逻辑非操作，进行相应的或操作
default Predicate<T> or(Predicate<? super T> other) {  
    Objects.requireNonNull(other);  
    return (t) -> test(t) || other.test(t);  
}

用来判断是否的函数接口，有一个test方法接受一个泛型对象，返回boolean类型。可以接受指定类型DoublePredicate IntPredicate LongPredicate以及一个BiPredicate 他接受两个参数，同时返回boolean类型，方法签名为``(T, U) -> boolean`；

package java.util.function;

@FunctionalInterface
public interface BiPredicate<T, U> {
    boolean test(T t, U u);
}

在Stream 中的filter 中要求提供一个Predicate类型的函数接口。

List<String> list = List.of("abc", "xyz", "12", "pq", "abcd", "1234");  
List<String> collect = list.stream().filter(s -> s.length() > 3).collect(Collectors.toList());  
System.out.println(collect);

Stream<T> filter(Predicate<? super T> predicate);

Function接口

package java.util.function;

@FunctionalInterface
public interface Function<T, R> {
    R apply(T t);
}

// 对两个Function进行组合，先执行before中的函数apply，然后将结果传递给调用对象作为函数的参数
default <V> Function<V, R> compose(Function<? super V, ? extends T> before) {  
    Objects.requireNonNull(before);  
    return (V v) -> apply(before.apply(v));  
}

// 对两个Function进行组合，先执行调用对象的apple方法，然后将结果传递给after作为函数的参数
default <V> Function<T, V> andThen(Function<? super R, ? extends V> after) {  
    Objects.requireNonNull(after);  
    return (T t) -> after.apply(apply(t));  
}

// 不知道有什么用
static <T> Function<T, T> identity() {  
    return t -> t;  
}

Function是对实例进行处理转换的接口，apply接受一个泛型T对象，返回一个泛型R对象。 在Stream 中的map 和 flatMap 中要求提供一个Function类型的函数接口。

<R> Stream<R> flatMap(Function<? super T, ? extends Stream<? extends R>> mapper);
<R> Stream<R> map(Function<? super T, ? extends R> mapper);

同时我们还有一个BiFunction接口，可以接受两个泛型对象的参数。

package java.util.function;

@FunctionalInterface
public interface BiFunction<T, U, R> {
    R apply(T t, U u);
}

List<String> list = List.of("abc", "xyz", "12", "pq", "abcd", "1234");  
String reduce = list.stream().reduce("list is: ", (a, b) -> a + "-" + b, (a, b) -> null);  
System.out.println(reduce);

// stream 中reduce源码
<U> U reduce(U identity,  
             BiFunction<U, ? super T, U> accumulator,  
             BinaryOperator<U> combiner);

Consumer接口

package java.util.function;

@FunctionalInterface
public interface Consumer<T> {
    void accept(T t);

	default Consumer<T> andThen(Consumer<? super T> after) {  
    Objects.requireNonNull(after);  
    return (T t) -> { accept(t); after.accept(t); };  
}
}

consumer用来消费对象，他有一个accept方法，只接受一个泛型T对象，没有返回值。

List<String> list = List.of("abc", "xyz", "12", "pq", "abcd", "1234");  
list.forEach(System.out::println);

// Iterable 对forEach的实现，接受一个Consumer类型的函数接口
default void forEach(Consumer<? super T> action) {  
    Objects.requireNonNull(action);  
    for (T t : this) {  
        action.accept(t);  
    }  
}

他有一个默认的andThen方法将两个Consumer组合，然后在进行消费。

Consumer<String> consumer1 = s -> System.out.println(s + " abc");  
Consumer<String> consumer2 = s -> System.out.println(s + " 123");  
// 在调用andThen后他也是返回一个Consumer接口  
consumer1.andThen(consumer2).accept("laelaps");

他还有个一个类似的接口BiConsumer<T, U>接受两个泛型T，U参数。

HashMap<String, Integer> map= new HashMap<>();  
map.put("A", 1);  
map.put("B", 2);  
map.forEach((a,b) -> System.out.println(a +  "- " + b));

// HashMap 中的forEach实现，使用了BiConsumer
public void forEach(BiConsumer<? super K, ? super V> action) {  
    Node<K,V>[] tab;  
    if (action == null)  
        throw new NullPointerException();  
    if (size > 0 && (tab = table) != null) {  
        int mc = modCount;  
        for (Node<K,V> e : tab) {  
            for (; e != null; e = e.next)  
                action.accept(e.key, e.value);  
        }  
        if (modCount != mc)  
            throw new ConcurrentModificationException();  
    }  
}
接口名称 方法名称 方法签名
Consumer accept (T) -> void
DoubleConsumer accept (double) -> void
IntConsumer accept (int) -> void
LongConsumer accept (long) -> void
ObjDoubleConsumer accept (T, double) -> void
ObjIntConsumer accept (T, int) -> void
ObjLongConsumer accept (T, long) -> void

UnaryOperator接口

@FunctionalInterface  
public interface UnaryOperator<T> extends Function<T, T> {  

    static <T> UnaryOperator<T> identity() {  
        return t -> t;  
    }  
}

UnaryOperator 扩展了Function接口，所以他具有Function的全部功能，他接受的类型和返回的类型都是同一种类型，所以他的可以用于单个操作数。

// 在List的replaceAll中有用到UnaryOperator接口
List<String> list = new ArrayList<>(List.of("abc", "xyz", "12", "pq", "abcd", "1234"));  
list.replaceAll(s -> "laelaps: " + s);  
System.out.println(list);

default void replaceAll(UnaryOperator<E> operator) {  
    Objects.requireNonNull(operator);  
    final ListIterator<E> li = this.listIterator();  
    while (li.hasNext()) {  
        li.set(operator.apply(li.next()));  
    }  
}

BinaryOperator接口

@FunctionalInterface  
public interface BinaryOperator<T> extends BiFunction<T,T,T> {  

     public static <T> BinaryOperator<T> minBy(Comparator<? super T> comparator) {  
        Objects.requireNonNull(comparator);  
        return (a, b) -> comparator.compare(a, b) <= 0 ? a : b;  
    }  
  
    public static <T> BinaryOperator<T> maxBy(Comparator<? super T> comparator) {  
        Objects.requireNonNull(comparator);  
        return (a, b) -> comparator.compare(a, b) >= 0 ? a : b;  
    }  
}

BinaryOperator 扩展了BiFunction接口，在BiFunction中三个泛型都是T，意味着他接受两个相同类型的参数，在对参数处理后，返回一个相同类型的返回值。

// map中的merge 接受一个key,一个value, 一个BinaryOperator类型的函数，当key不存在直接是put(key, value)；如果存在，则会调用BinaryOperator类对值进行处理过在进行更新操作。
HashMap<String, Integer> map= new HashMap<>();  
map.put("A", 1);  
map.put("B", 2);  
map.merge("A", 3, (oldValue, newValue) -> newValue + 100 + oldValue);  
map.forEach((a,b) -> System.out.println(a +  "- " + b));

在BinaryOperator 中有两个静态方法minBy maxBy ，很明显这两个函数接受一个comparator类型的lambda函数，对两个T类型的参数进行比较后返回最小的那个或者最大的那个。

Comparator接口

package java.util;

@FunctionalInterface  
public interface Comparator<T> {
	int compare(T o1, T o2);
}

他并不是在java.util.function包中，而是在java.util中，同时他也是一个函数式接口，他接受两个泛型T的参数，返回一个int类型，为一个负整数，第一个参数小于第二个；返回0，第一个参数等于第二个；为一个正整数，第一个参数大于第二个。

Runnable接口

package java.lang;

@FunctionalInterface  
public interface Runnable {  
    public abstract void run();  
}

Runable 是实现Thread必须要实现的一个函数式接口。

new Thread(() -> System.out.println("laelaps")).start();
