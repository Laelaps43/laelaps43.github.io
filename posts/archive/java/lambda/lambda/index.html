是什么？

Java的lambda表达式是一个可传递的代码块，可以在之后执行一次或者多次。 我们都知道Java是一种面向对象的语言，所以我们如果想传递一个代码款，我们必须构造一个对象，然后在对象中实现一个方法包含所需要的代码块。

// 实现一个内部类来传递需要实现的代码
Thread thread1 = new Thread(new Runnable() {
            @Override
            public void run() {
                System.out.println("abc");
            }
        });

// 使用lambda表达式
new Thread(() -> System.out.println("xyz")).start();

语法

Java中lambda表达式的形式：参数 ->（箭头）{代码块}/一个表达式。

  1. 如果需要实现的方法代码很多，可以将代码放入一个{}中
 // 使用一个代码块实现，将代码放入一个{}中
 Arrays.sort(arr, (String str1, String str2) -> {
    if(str1.length() < str2.length()) return -1;
    else if(str1.length() == str2.length()) return 0;
    else return 1;
});
  1. 如果lambda表达式可以推导出参数的字符串，我们可以参数的类型省去，但是值得注意的是Java是一种强类型的编程语言，他需要明确的知道参数的类型，这里可以省去的原因是Java编译器可以从上下文中推到出来。
 Arrays.sort(arr, (str1, str2) -> {
    if(str1.length() < str2.length()) return -1;
    else if(str1.length() == str2.length()) return 0;
    else return 1;
});
  1. 如果lambada表达式只有一个参数，我们可以同时将参数类型以及()省去。
List<String> strings = List.of("abc", "xyz", "wert");
strings.forEach((String s) -> System.out.println(s));
// 同时省去参数类型以及()
strings.forEach(s -> System.out.println(s));
  1. 如果lambda表达式中没有参数，我们也需要同时提供空括号
new Thread(() -> System.out.println("xyz")).start();
  1. 如果只是一个表达式，编译器会自动返回值，如果使用大括号包围，则需要手动的指明返回值；并且返回值的类型可以自动推到。

函数式接口

引用

方法引用（method reference）

在一般情况下，我们都是自己写lambda的body部分，但是在有一些情况下，在Java中有一些方法已经实现了类似的操作，我们只需要引用他，引用的条件是方法的签名必须要接口恰好一致。 使用::来表示引用运算符

静态方法引用

Class::staticMethod 使用静态方法引用实时将所有的方法都传递到静态方法中 Math::pow -> (x, y) - > Math.pow(x,y)

成员方法引用

object::instanceMethod System.out::println() -> x -> System.out.println(x) System.out为对象，println()为对象中的实例方法

类的对象的实例方法

Class::instanceMethod 他接受的第一个参数成为方法的隐式参数，及使用第一个参数去调用这个方法。 String::compareToIgnoreCase -> (x, y) -> x.compareToIgnoreCase(y)

注意

  1. 如果有多个重名的重载方法，编译器会从找出你所指的那个方法。
  2. 在方法应用时也可以使用this 和 super。

构造器引用

构造器的引用跟方法的引用类似，区别在于方法名为new，如果有多个构造器，他会从上下文中推倒相对应的来构造器。

List<String> strings = List.of("abc", "xyz", "wert");
List<Person> collect = strings.stream().map(Person::new).toList();

我们没有办法将一个stream流中的数据创建一个对应类型的数组保存，因为Java中不可能创建一个泛型数组new T[]，但是我们可以通过stream.toArray来生产一个数字，但是他默认返回的是一个Object[]类型的数组，但是他同时接受一个指定类型的lambda函数，我们可以提供一个相应类型的构造函数。

变量作用域

lambda 有三部分：一个代码块，参数，以及自由变量的值。 自由变量的值是可以重调用这个方法中捕获的，他获取的变量必须是事实最终变量（effectively final）；就是确保这个变量初始化后不会再为他赋值（如下面的abc）。

public static void main(String[] args){
    String abc = "abc";
    new Thread(() -> {
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
        System.out.println("sleep 1000 millis, abc is " + abc);
    }).start();
    System.out.println("main thread quite");
}

在一个lambda表达式中不能有两个同名的局部量

Path first = Path.of("/usr/bin");
Comparator<String> comp = (first, second) -> first.length() - second.length();

this

如果一个lambda中使用了this，那么this这是代表调用这个lambda函数的这个方法。

总结

我们使用lambda函数是为了他的延迟执行（deferred execution），是当有需要的时候才会执行，而不需要的时候并不消耗额外的CPU。如果我们使用内部类的方法，那么无论什么时候我们都需要先去new一个对象，如果我们并不需要去使用这个内部类的实现，其实在浪费CPU、内存。采用lambda函数可以避免这种情况。
