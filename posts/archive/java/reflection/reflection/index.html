Class类

Class类是真实存在的，他在java.lang下，他跟普通的类还是有很多不一样的；在Java中一切都是对象，而在java中有两种对象，一种是我们创建的实例对象，而另外一种是有JVM以及类加载器创建的Class对象，其实也就是说Class对象并不是我们自己创建的，他应该由JVM来创建；

Class保存的是什么信息呢？他用来表示一般类在运行时的类型信息，我们创建的实例对象在JVM中都应该根据Class对象来创建；而他被用来进行RTTI（Run-Time Type Identification）运行类型识别，在运行时识别一个对象的类型和类的信息。

在Java中每一个类都有一个Class对象，每当编译一个新类就产生一个 Class对象，基本类型 (boolean, byte, char, short, int, long, float, double)有 Class对象，就连关键字 void也有Class对象（void.class）。

前面说过Class对象只能由JVM及类加载器来创建，所有Class只有一个私有构造器；Java中的所有类都是懒加载到JVM中的，只有当需要的时候才会被加载；在类加载过程中，类加载器会检查这个类的Class对象是否存在，如果未被加载，那么类加载器会根据类的全限定名查找.class文件，随后类加载器进行验证等操作，生产Class对象，根据Class对象来生成实例对象。

获取Class对象

Class.forName

package org.example;  
  
public class Ron {  
  
    static {  
        System.out.println("I am Ron");  
    }  
}


package org.example;  
  
public class Main {  
    public static void main(String[] args) throws ClassNotFoundException {  
        Class<?> ron = Class.forName("org.example.Ron");  
    }  
}

static{}会被类第一次被加载的时候被执行，所以new或forName都会判断类是否被加载，forName获取Class对象时并不需要去拥有该类型的对象，而只需要全限定名就可以获取到对应的Class对象。

getClass

package org.example;  
  
public class Main {  
    public static void main(String[] args){  
//        Class<?> ron = Class.forName("org.example.Ron");  
        Ron ron = new Ron();  
        Class<?> ronClass = ron.getClass();  
    }  
}

Class对象在new时就已经被加载到了JVM中，所有getClass并不会重新加载。

.class

package org.example;  
  
public class Main {  
    public static void main(String[] args){  
//        Class<?> ron = Class.forName("org.example.Ron");  
//        Ron ron = new Ron();  
//        Class<?> ronClass = ron.getClass();  
        Class c1 = int.class;  
        Class c2 = Integer.class;  
  
        Class c3 = Ron.class;  
    }  
  
}

在使用.class方法中，并不会运行static{}，因为不会自动的创建Class对象，类加载被延迟到了对静态方法或者非常数静态域首次引用时才执行。

唯一性

在内存中每一个类只有一个与之对应的Class对象来描述基本类型信息，所以无论我们创建多少个Class对象，最后指向的都是同一个Class对象。

反射

在Java中如果我们需要去执行一个对象的方法，我们可以有静态加载或者是动态加载一个对象，然后再去执行他的方法；

何为静态加载和动态加载？如果我们在编译时就指定了对象的类型，那么为静态加载；而在JVM运行时才确定对象类型为动态加载。

package org.example;  
  
import java.lang.reflect.Constructor;  
import java.lang.reflect.Method;  
  
public class Main {  
    public static void main(String[] args) throws Exception {  
	    // 静态加载
        Ron ron = new Ron();  
        ron.callMe();  
  
        // 动态加载  
        Class<?> ronClass = Class.forName("org.example.Ron");  
        Constructor<?> constructor = ronClass.getConstructor();  
        Method callMe = ronClass.getMethod("callMe");  
  
        Object ronObj = constructor.newInstance();  
        callMe.invoke(ronObj);  
    }  
}

在这里我们只用更改一下全限定名称和拥有callMe方法的类就可以继续运行。

反射就是在程序运行期间，我们都可以知道这个类所有的方法和属性，我们可以使用反射去构建或执行需要的方法；在Java由Class类和java.lang.reflect包来实现反射操作；反射可以让我们的程序解耦。

反射操控构造器

package org.example;  
  
  
import lombok.Data;  
  
@Data  
public class Student {  
    private Integer id;  
    private String name;  
  
    private Student(String name){  
       this(1, name);  
    }  
  
    public Student() {  
        super();  
    }  
  
    public Student(Integer id, String name) {  
        super();  
        this.id = id;  
        this.name = name;  
    }  
  
    public static void eat() {  
        System.out.println("student can eat");  
    }  
}

package org.example;  
  
import java.lang.reflect.Constructor;  
  
public class Main {  
    public static void main(String[] args) throws Exception {  
        Class<?> studentClass = Class.forName("org.example.Student");  
  
        // 通过Class中的newStance来创建对象 这种方式被抛弃  
        Object student1 = studentClass.newInstance();  
  
        // 通过Constructor的newInstance来构建对象  
        Constructor<?> constructor = studentClass.getConstructor(Integer.class, String.class);  
        Object student2 = constructor.newInstance(11, "Ron");  
  
        Constructor<?> constructor1 = studentClass.getDeclaredConstructor(String.class);  
        // 如果不开启，会产生IllegalAccessException异常  
        constructor1.setAccessible(true);  
        Object student3 = constructor1.newInstance("Ron");  
    }  
}

在Class中有getConstuctor(Class<?>... parameterTypes)来根据给定的paramterTypes变长数组来获取对应的公共方法，如果不给定参数，获取的则是无参构造器；使用getConstructors获取指定类中的所有公共方法；而获取私有方法使用getDeclaredConstructor，即添加了Declared方法，在获取属性和方法也是这个规则；使用getDeclaredConstructors可以获取到所有的构造方法。

Construcor常用方法

String getName() //获取构造器名； 
Class getDeclaringClass() //获取构造器所属的类型； 
Class[] getParameterTypes() //获取构造器的所有参数的类型 class； 
Class[] getExceptionTypes() //获取构造器上声明的所有异常类型 
class； T newInstance(Object… initargs) //通过构造器反射对象调用构造器。
setAccessible(true) //使得私有构造器打开 
<T extends Annotation> T getAnnotation(Class<T> annotationClass) //如果存在该元素的指定类型的注解，则返回这些注解，否则返回 null。 Annotation[] getDeclaredAnnotations() //返回直接存在于此元素上的所有注解。

反射获取属性

public class Main {  
    public static void main(String[] args) throws Exception {  
        Class<?> studentClass = Class.forName("org.example.Student");  
  
        // 获取名称为name的属性  
        Field name = studentClass.getDeclaredField("name");  
  
        // 获取所有的属性，不包括继承  
        Field[] declaredFields = studentClass.getDeclaredFields();  
    }  
}

当然还存在getField获取指定名称的公共属性，getFields获取全部的公共属性。

设置属性

public class Main {  
    public static void main(String[] args) throws Exception {  
        Class<?> studentClass = Class.forName("org.example.Student");  
  
        // 获取名称为name的属性  
        Field name = studentClass.getDeclaredField("name");  
        Field id = studentClass.getDeclaredField("id");  
  
        // 创建一个实例对象  
        Student student = (Student)studentClass.getConstructor().newInstance();  
  
        // 设置访问权限,可以访问private修饰的属性,否则抛出:IllegalAccessException  
        name.setAccessible(true);  
        id.setAccessible(true);  
  
        //给该对象设置属性值.obj-实例对象 value-属性的值  
        name.set(student, "Ron");  
        id.set(student, 12);  
  
        System.out.println(student);  
  
    }  
}

反射获取成员方法

  
public class Main {  
    public static void main(String[] args) throws Exception {  
        Class<?> studentClass = Class.forName("org.example.Student");  
  
        // 获取指定名称的方法  
        Method getName = studentClass.getMethod("getName");  
        Method setName = studentClass.getMethod("setName", String.class);  
        Method setId = studentClass.getDeclaredMethod("setId", Integer.class);  
        // 静态方法  
        Method eat = studentClass.getMethod("eat");  
  
        // 获取所有的方法  
        Method[] declaredMethods = studentClass.getDeclaredMethods();  
  
        // 执行方法  
        Object student = studentClass.getDeclaredConstructor().newInstance();  
        setName.invoke(student, "Ron");  
        setId.invoke(student, 12);  
  
        String name = (String)getName.invoke(student);  
        System.out.println(name);  
        // 执行静态方法  
        eat.invoke(null);  
        eat.invoke(student);  
    }  
}

在使用getMethod(String name, Class<?>... parameterTypes)有一个parameterTypes变长数字，他根据parameterTypes来查找同名的方法。
