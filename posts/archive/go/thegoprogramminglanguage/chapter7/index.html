前言

Go中通过接口支持鸭子类型 If it looks like a duck, swims like a duck, and quacks like a duck, then it probably is a duck.

What?

接口类型是对其他类型行为的概括和抽象，一个接口定义了一套方法，如果一个类型需要实现该接口，那么需要实现接口中定义的所有方法，所以对于一个接口对象，我们不知道他是什么，但是我们可以知道他能做什么，及有那些方法，及接口是一种抽象的类型，他并没有暴露他所含有的数据的布局和内部结构，而是只有操纵他的方法。 接口要求类型包含了与签名和行为与接口类的方法一致的方法；实现了接口类型的任何方法都可以等于这个接口类型。 接口类型可以嵌套其他接口类型，这个新的接口类型需要去实现所有嵌套接口的类型。

package main

import "fmt"

type Person struct{
	Name string
	Age int
}

type Lae interface {
	play()
}

func (p Person)play(){
	fmt.Println(p.Name + " play")
}

func main() {
	lae := new(Person)
	lae.Name = "Ron"
	lae.Age = 18
	var inter Lae = lae
	inter.play()
}

空接口类型

空接口类型interface{}，不包含任何的方法，所有的类型都实现了这个空接口，任何值都可以复制给空接口类型。

获取值

我们将类型的值赋值给空接口，但是空接口不包含任何方法，我们无法从接口中获取出原来的实际值。 在Go 1.18之后，Go 将interface{}全部替换成any。

接口值

一个接口类型的值有两个部分：一个具体类型（动态类型）和一个该类型的值（动态值）。 接口的初始化为一个零值，动态类型和动态值动态值都是nil；一个接口是否为nil根据他的动态类型来判断。

var w io.Writer
w = os.Studout
w = nil

在编译时我们无法知道一个接口值的类型是会是什么，所以我们通过接口来调用必须使用动态分发，所以编译器必须通过类型描述来拿到对应对象的地址，然后间接调用该方法地址。

w.wirte([]byte("hello"))
// 等于
os.Studout.Write([]byte("hello"))

接口值可以用==和!=操作来做比较，如果两个接口值的动态类型是相同的并且动态值相等，那么两个接口值相等；如果动态值是不可比较的，那么这个比较将会崩溃。

error 接口

error类型是一个接口类型。

type error interface {
	Error() string
}

创建一个error类型最简单的方法是调用errors.New，返回一个包含指定错误的error实例，但是这并不是最常用的，通常更多使用的是fmt.Errorf，他提供了格式化字符串的功能。

类型断言

类型断言用在接口值上，用来判断操作数的动态类型是否满足所自定的断言类型，类似于x.(T)。 我们通常用他来判断一个接口是否为一个特定的类型。 在使用断言时，如果我们将断言的结果赋值给两个变量，那么断言不会panic，会多返回一个布尔类型的返回值来指示断言是否成功。 断言的T有两种可能，一种是为具体的类型，类型断言会检查x的动态类型是否为T，如果正确，那么断言的结果就是x的动态值，类型为T。 如果为一个接口类型，那么断言的是判断x的动态类型是否满足于T；如果满足，只是将接口值的类型变为T，接口值的动态类型和动态值没有改变；从一个接口到另外一个接口，他的方法的数量应该是变多的。

func main() {
	var w io.Writer
	w = os.Stdout
	fmt.Printf("%T, %q\n", w, w)
	var f1 *os.File
	f1, ok := w.(*os.File)
	if !ok{
		fmt.Println("panic")
	}
	fmt.Printf("%T, %q\n", f1, f1)

	var f io.ReadWriter
	f = w.(io.ReadWriter)
	fmt.Printf("%T, %q\n", f, f)
}

类型分支

接口的风格：

  • 在接口上的方法满足了这个接口的具体类型之间的相似性，但是隐藏了具体类型的布局和各自特有的功能，强调跟多的是方法，不是具体类型。
  • 利用接口值可以容纳各种具体类型的能力，接口是这些基本类型的联合。注重的接口中的类型，而不是接口中的方法和信息的隐藏，称为discriminated union。 我们知道switch可以用来简化一系列的if-else语句，如果我们需要判断一个接口的类型，根据断言来区分，我们可能会写一系列的if-else，但是Go给我提供了类型分支（type switch）来简化一序列的类型断言的if-else语句；他跟普通的switch语句类似。
func Quote(y any)string{
	if y == nil {
		return "NIL"
	}else if _, ok := y.(int); ok{
		return fmt.Sprintf("int %d",y)
	}else if _, ok := y.(uint); ok{
		return fmt.Sprintf("uint %d",y)
	}else if b, ok := y.(bool); ok{
		if b {
			return "TRUE"
		}
		return "FALSE"
	}else if s, ok := y.(string); ok{
		return fmt.Sprintf("strings %s",s)
	}else{
		panic(fmt.Sprintf("unexcept value %T %v", y, y))
	}
}

func QuoteTypeSwitch(y any)string{
	// 如果我们的case并不使用x，那么我们可以将x省去
	switch x := y.(type){
	case nil: return "NIL"
	case int: return fmt.Sprintf("int %d", x)
	case uint: return fmt.Sprintf("uint %d", x)
	case string: return fmt.Sprintf("strings %s", x)
	case bool :
		if x {
			return "TRUE"
		}
		return "FALSE"
	default:  panic(fmt.Sprintf("unexcept value %T %v", x, x))
	}
}

func main() {
	s := Quote("abc")
	fmt.Println(s)

	s1 := QuoteTypeSwitch(1)
	fmt.Println(s1)
}
