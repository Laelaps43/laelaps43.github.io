什么是容器？

容器是镜像运行的实例，我们可以从一个镜像中运行多个容器，容器比传统的虚拟机更快和更轻量级；在采用虚拟机时，我们的虚拟机的软件要将物理资源进行划分，然后分配给每一个虚拟机，他是一种居于硬件虚拟化；而在容器时代，我们的物理资源全部分配给操作系统，我们的容器引擎想操作系统申请资源，每一个容器是一个独立的相互隔离的进程，他可以自动的调整资源的大小，容器是一种居于操作系统的虚拟化。

启动容器

docker container run <options> <im-age>:<tag> <app>
// 启动一个ubuntu
docker container run -it ubuntu:20.04 /bin/bash
// 或者省去container 
docker run -it ubuntu:20.04 /bin/sh

当运行docker run时，docker会根据指定的镜像的标签在本地查找，如果查找不到则自动进行pull。然后运行命令指定的程序。

  • -t是分配一个伪终端将（pseudo-tty）绑定到容器的标准输入上。
  • -i是让容器的标准输入保存打开。
  • 退出当前终端使用exit()，容器会停止运行，因为这时候终端是容器唯一运行的程序。 docker使用docker container start来启动一个已经停止的容器；使用docker container restart来将一个运行中的容器重新启动它。

查看容器

docker使用docker container ls来查看当前正在运行的容器，在ls之后添加-a来查看全部的容器，包括已经退出的容器。

删除容器

我们在查看所有容器后，会发现有许多停止运行的容器，使用docker container rm可以将已经停止运行的容器删除。

$ docker container ls -a
CONTAINER ID   IMAGE                      COMMAND                  CREATED          STATUS                      PORTS                                            NAMES
7f87eac41019   ubuntu:20.04               "/bin/bash"              13 minutes ago   Exited (0) 6 minutes ago                                                     condescending_jennings
f412d9c4ff7e   ubuntu:20.04               "/bin/sh"                14 minutes ago   Exited (0) 7 minutes ago
$ docker container rm 7f87eac41019 f412d9c4ff7e
7f87eac41019
f412d9c4ff7e

守护态运行

在之前我们都是的容器都是将输入结果输出到当前的数组的终端上，如果我们想让容器运行在后台上，可以在docker run -d ..添加-d之上，容器在后台运行。

$ docker run -d ubuntu:20.04 /bin/sh -c "while true; do echo hello world; sleep 1; done"

我们在上面之中运行了-d，并同时不断的打印出hello world，我们可以使用docker container logs 来查看当前标准输出。

$ docker container logs 2833a08
hello world
hello world
hello world
hello world
hello world

停止容器

在一般情况下，我们使用容器是需要就创建，用完删除，我们可以使用docker contaienr <container> -f来删除一个正在容器，但是这种方法并不友好，容器被毫无征兆的删除，docker 更推荐的是使用docker container stop <container>之后，然后再使用docker contaienr rm <container>删除。 docker container rm -f vs docker container stop

  • 使用后者他会想进程发送容器要停止的警告信息，然后在多一段时间后停止容器，而前者则是直接删除容器。
  • 后者会先发送SIGTERM型号，容器可以收到，可以处理退出的程序；然后发送SIGKILL信号来直接终止进程，前者直接发送SIGKILL信号。

进入容器

在使用-d后，容器启动后进入后台，我们需要重新进入容器怎么办？ 使用docker attach <container>或者docker exec <container> 他们都可以重新进入容器，但是前者使用exit()后会终止容器，将容器停止，而后者不会。
