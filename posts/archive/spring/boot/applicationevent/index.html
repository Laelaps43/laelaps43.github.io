事件监听

Spring中的事件监听是观察者模式的一种实现，在观察者模式当中，应当有主题、发布者、订阅人；所有在Spring的事件中存在事件（ApplicationEvent）、监听器（ApplicationListener）和事件发布三部分。

事件用于在松耦合的组件之间交换信息，发布者和订阅者之间没有直接耦合，使得修改发布者并不会影响订阅者，反之也成立。

监听事件

监听事件是消息传递的对象，用来传递事件消息，在Spring中我们去继承ApplicationEvent来定义一个事件。

public class MyEvent extends ApplicationEvent {  
    private String name;  
  
    private int age;  
  
    public MyEvent(Object source, String name, int age) {  
        super(source);  
        this.age = age;  
        this.name = name;  
    }  
}

事件发布

在定义事件后，我们需要发布事件，我们可以使用ApplicationContext和ApplicationEventPublisher两者来发布事件；前者去继承了后者。

@Service  
public class MyEventService {  
    @Autowired  
    ApplicationContext applicationContext;  
    @Autowired  
    ApplicationEventPublisher applicationEventPublisher;  
  
    public void myEvent(){  
        System.out.println("发布事件......");  
        applicationContext.publishEvent(new MyEvent(this, "laelaps", 12));  
        System.out.println("事件发布完毕。");  
    }  
      
    public void myEvent2(){  
        System.out.println("发布事件......");  
        applicationEventPublisher.publishEvent(new MyEvent(this, "laelaps", 12));  
        System.out.println("事件发布完毕。");  
    }  
}

事件监听

在发布了事件后，我们需要监听者去监听发布的事件；在Spring中有两种方式来定义监听对象；

  1. 使用@EventListener注解来定义监听对象。
  2. 实现ApplicationListener接口。

这两种都需要向IOC容器中注册，因为要让Spring去托管这些Bean。

使用接口来定义监听器：

@Component  
public class MyEventListener implements ApplicationListener<MyEvent> {  
  
    public void onApplicationEvent(MyEvent event) {  
        System.out.println("MyEventListener start......");  
        System.out.println(event);  
    }  
}

使用注解来定义监听器：

@Component  
public class MyEventListener{  
  
    @EventListener(classes = {MyEvent.class})  
    public void onApplicationEvent(MyEvent event) {  
        System.out.println("MyEventListener start......");  
        System.out.println(event);  
    }  
}

一个方法的注解可以监听多个事件或者不使用参数来定义。

@EventListener(classes={ContextStartedEvent.class, ContextRefreshedEvent.class})

在注解中还有一个condition属性，他接受一个SpEL表达式，用来表明在那些情况下可以触发监听器。

如果定义了多个监听器的话，那么他们都会被触发。

异步

在一般情况下，Spring事件是同步的，意味着在发布者发布后，只有当监听器处理之后才能继续运行，其他的时间下他是被阻塞的；我们可以在监听器中添加@Async注解来实现监听事件来以异步的方式运行；在使@Async生效，需要使用@EnableAsync注释SpringBootApplication类。
